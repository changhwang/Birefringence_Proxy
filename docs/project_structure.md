# Project Structure

This document records the current top-level storage conventions for the birefringence analysis workspace.

## 1. Intent

The repository is organized so that:

- `data` contains source data and preview inputs
- `configs` contains editable analysis rules and presets
- `analysis_outputs` contains generated artifacts only
- `docs` records methodology and workflow decisions

## 2. Config layout

Dataset-level runtime configuration:

- [analysis_manifest.json](../configs/datasets/analysis_manifest.json)

ROI preset configurations:

- [horizontal_center_3split.json](../configs/roi_presets/horizontal_center_3split.json)
- [refined_grid_3x3_v1.json](../configs/roi_presets/refined_grid_3x3_v1.json)

## 3. Output layout

- [analysis_outputs/rotation_calibration](../analysis_outputs/rotation_calibration)

- [safe_region_baseline](../analysis_outputs/roi_review/safe_region_baseline)

Pipeline outputs are grouped under a dedicated run root:

- [pipeline_runs/current](../analysis_outputs/pipeline_runs/current)

Inside the current run:

- [signals](../analysis_outputs/pipeline_runs/current/signals)
- [fits](../analysis_outputs/pipeline_runs/current/fits)
- [qc](../analysis_outputs/pipeline_runs/current/qc)
- [summary](../analysis_outputs/pipeline_runs/current/summary)

## 4. Practical rule

If a file is hand-edited, it should live under `configs` or `docs`.

If a file is generated by a script, it should live under `analysis_outputs`.

If a file is acquired from the camera or copied as an input reference, it should live under `data`.
